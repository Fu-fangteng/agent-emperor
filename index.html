Agent 皇帝（Agent Emperor）

多 Agent 协作框架

皇帝不是亲自搬砖的人，而是低频编排器：看一眼奏折、复制 prompt、开新对话传旨。规划大臣（planner）、开发大臣（developer）、审查大臣（reviewer）按 STATUS 接力，所有交接都落在文件总线上。

四个核心概念

        真实示例，不读运行时文件
      

      

        

          
官职与工具解耦

          
team.yaml 是唯一事实源：agent 名字、工具、role 和 handoff 都从这里生成。下方是本页三角色教学编制；本仓库实际编制见上方说明。

          
roles: [planner, developer, reviewer]
agents:
  - name: cc
    roles: [planner, reviewer]
  - name: codex
    roles: [developer]

        

        

          
文件总线即奏折

          
规划、实现、审查都写进总线文件；聊天只负责把下一卷奏折交出去。

          
docs/agent-collaboration/
  phases/v1/
    requirements.md
    plan.md
    handoff.md
    review.md

        

        

          
一位大臣一个对话

          
developer 和 reviewer 分开窗口运行，避免同一上下文里自己审自己。

          
developer:
  read: plan.md
  write: handoff.md
reviewer:
  read: handoff.md + diff
  write: review.md

README / 配置怎么读

        上手顺序：README → team.yaml → 生成文件
      

      

        

          
README 负责说明人怎么接力

          
README 应写清三步：配置团队、生成 agent 指令、按 handoff.md 的 STATUS 串行接力。人不需要记所有规则，只要看状态、复制 prompt、开新对话。

          
1. 编辑 team.yaml
2. python3 core/generate.py --team team.yaml --target .
3. 从 START_HERE.md + handoff.md 开始接力

        

        

          
配置文件负责约束 AI 怎么行动

          
team.yaml 定义角色、工具、状态机、文件所有权和身份锚点；生成器把这些规则落成 AGENTS.md / CLAUDE.md 等工具侧指令。

          
anchor:
  style: chunni
bus:
  ownership:
    - { file: handoff.md, owner: developer }
    - { file: review.md, owner: reviewer }
